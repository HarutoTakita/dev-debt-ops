# 粒度間の整合・定期再計測・クイズ粒度の最適化

> 親トラッキング: [057 多粒度計測の拡張](./057-multi-granularity-code-debt-rollout.md)（C）から切り出し。

## 概要

多粒度計測（機能 / フォルダ / ファイル / クラス / 関数）が出揃った後の**横断的な整合と運用**を扱う。

## 残タスク

- [ ] 粒度間ロールアップの一貫性（関数 → クラス → ファイル → フォルダ/機能）を保証する集約規約と検証。
- [ ] 定期再計測（粒度別スナップショットのトレンド。`debt_trend_points` の粒度対応）。
- [ ] クイズ / 学習の粒度対応の最適化（クラス/関数単位の設問は過剰になり得るため、適切な粒度の検討）。

## 完了条件

- 各粒度の値がロールアップとして整合し、粒度別トレンドが取得でき、クイズ/学習が適切な粒度で運用される。

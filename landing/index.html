コードの“わかってなさ”を、
測って返す。

理解度をクイズで実測（git blame 非依存・単独開発でも計測可）し、 学習で返済する閉ループ。技術負債はリファクタ PR で返す。 理解負債と技術負債を、一枚のダッシュボードで。

無料で始める 仕組みを見る

負債は「コード」だけじゃない

書いた本人が抜け、AI 生成が増えるほど「誰も中身を理解していないコード」が積み上がる。 これは技術負債とは別の理解負債。放置すると変更速度も品質も落ちる。

2つの負債を、可視化して返済する

理解負債

  • 可視化｜理解度マップ — 誰がどこを理解しているかを星図で俯瞰。
  • 返済｜クイズと学習 — クイズで実測し、学習プランで埋める。合格で理解度↑。

技術負債

  • 可視化｜コード品質マップ — 重複・複雑度・AI 生成痕跡を検知。
  • 返済｜コード改善 — AI がリファクタ案を生成し、返済 PR を作成。

実測 → 返済の閉ループ

① 解析（検知・理解度・機能） → ② クイズで実測 → ③ 学習で返済 → ④ リファクタ PR ↺

コード負債の検知は出発点ではなく「どの理解ギャップが緊急か」を示すホットスポット信号。 主役はあくまで理解度の実測と返済です。

blame 非依存の実測

推測ではなくクイズで能動計測。単独開発もコードを書かない PM も対象。

GitHub 連携で自動化

リポジトリを接続して解析。返済はそのまま GitHub PR に。

機能（feature）単位の単元

ファイルを機能に束ね、学習 → 確認クイズで理解を積み上げる。
